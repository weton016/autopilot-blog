## DATABASE (Supabase)

```sql
-- Posts
create table posts (
  id uuid primary key default gen_random_uuid(),
  slug text unique not null,
  title text not null,
  excerpt text not null,
  content text not null,
  cover_image_url text,
  funnel_stage text not null check (funnel_stage in ('TOP', 'MIDDLE', 'BOTTOM')),
  topic_cluster text,
  related_post_ids uuid[],
  keywords text[],
  primary_keyword text,
  keyword_volume int,
  keyword_difficulty int,
  meta_title text,
  meta_description text,
  og_image text,
  read_time_minutes int,
  published_at timestamptz default now(),
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);

-- Content creation jobs
create table content_jobs (
  id uuid primary key default gen_random_uuid(),
  scheduled_date date not null,
  funnel_stage text not null check (funnel_stage in ('TOP', 'MIDDLE', 'BOTTOM')),
  topic_hint text not null,
  primary_keyword text,
  keywords_to_target text[],
  keyword_volume int,
  keyword_difficulty int,
  status text not null default 'PENDING' check (status in ('PENDING', 'RUNNING', 'DONE', 'FAILED')),
  post_id uuid references posts(id),
  error_message text,
  created_at timestamptz default now(),
  executed_at timestamptz
);

-- Topic history (avoid repeating content)
create table topic_history (
  id uuid primary key default gen_random_uuid(),
  topic text not null,
  funnel_stage text not null,
  primary_keyword text,
  keywords text[],
  created_at timestamptz default now()
);

-- Cache of researched keywords (avoid unnecessary API re-queries)
create table keyword_cache (
  id uuid primary key default gen_random_uuid(),
  keyword text unique not null,
  volume int,
  difficulty int,
  cpc numeric,
  fetched_at timestamptz default now()
);

-- Indexes
create index posts_slug_idx on posts(slug);
create index posts_funnel_stage_idx on posts(funnel_stage);
create index posts_published_at_idx on posts(published_at desc);
create index content_jobs_scheduled_date_idx on content_jobs(scheduled_date);
create index content_jobs_status_idx on content_jobs(status);
create index keyword_cache_keyword_idx on keyword_cache(keyword);
```

## BUSINESS LOGIC

### FunnelStrategyService — Strategy Pattern

Implement the 60% TOP / 25% MIDDLE / 15% BOTTOM ratio.
For each week, compute how many posts of each stage to create based on `POSTS_PER_DAY * 7`, respecting the ratios and ensuring at least one of each type.

---

### Keyword Research (NEW — accelerates traffic)

The system must not generate posts on random topics. Before creating weekly jobs, it must run keyword research via the DataForSEO API to find terms with:

- Monthly search volume > 100
- Keyword Difficulty (KD) < 40 (low competition)
- Relevance to `BLOG_TOPIC`

Implement `KeywordResearchAdapter` that:

1. Uses OpenAI to generate a list of 30 seed keywords related to `BLOG_TOPIC` and the funnel stage
2. Sends those seeds to the DataForSEO API (`/v3/keywords_data/google_ads/search_volume/live`)
3. Filters and ranks by best opportunity (high volume + low difficulty)
4. Checks `keyword_cache` before calling the API (30-day TTL)
5. Returns the best keywords for `ScheduleWeeklyJobsUseCase` to use when creating jobs

Each created job must have `primary_keyword`, `keywords_to_target`, `keyword_volume`, and `keyword_difficulty` populated.

`GeneratePostUseCase` must use `primary_keyword` as the content anchor, ensuring it appears in the title, first paragraph, at least one H2 subheading, and the meta description.

---

### ScheduleWeeklyJobsUseCase

- Runs every Monday via cron
- Runs `KeywordResearchAdapter` to get volume- and difficulty-validated keywords
- Computes posts needed for the next `POST_CREATION_INTERVAL_DAYS` days
- For each post sets: `funnel_stage`, `topic_hint`, `primary_keyword`, `keywords_to_target`
- Queries `topic_history` to avoid repeating topics and keywords already used
- Persists `content_jobs` with `scheduled_date` spread across the week

### ExecuteDailyJobsUseCase

- Runs daily via cron
- Fetches jobs with `scheduled_date = today` and `status = PENDING`
- For each job: runs `GeneratePostUseCase` then `GenerateCoverImageUseCase`
- Updates job status (`RUNNING → DONE | FAILED`)

### GeneratePostUseCase

Uses `PromptBuilder` (Builder Pattern) to assemble a rich prompt containing:

- Business context (`BLOG_TOPIC` + `WEBSITE_URL`)
- Funnel stage with tone and objective instructions
- `primary_keyword` as mandatory content anchor
- Secondary keywords for natural inclusion
- Existing topics to avoid repetition

The generated post must include:

- SEO-friendly title with `primary_keyword`
- Excerpt (150–160 chars) with the keyword
- Markdown content (1500–2500 words)
- Keyword in the first paragraph, at least one H2, and the conclusion
- Meta title and meta description with the keyword
- Primary and secondary keywords
- Thematic cluster
- Related post IDs for internal linking
- Automatically calculated read time

After creation, record the topic and keyword used in `topic_history`.

---

### GenerateCoverImageUseCase (NEW — Google Gemini)

After creating the post, automatically generate a cover image using the Google Gemini API (`gemini-2.0-flash-preview-image-generation` or the latest available image model).

Flow:

1. Receives the title and excerpt of the newly created post
2. Uses `PromptBuilder` to build a descriptive, modern image prompt related to the post theme — no text in the image, professional digital illustration style
3. Calls the Gemini API to generate the image
4. Uploads the image to Supabase Storage via `SupabaseStorageAdapter` in the `post-covers` bucket, named `{post-slug}.webp`
5. Updates the post’s `cover_image_url` and `og_image` with the public image URL

Example Gemini API call:

```ts
import { GoogleGenerativeAI } from "@google/generative-ai";

const genAI = new GoogleGenerativeAI(process.env.GEMINI_API_KEY!);
const model = genAI.getGenerativeModel({
  model: "gemini-2.0-flash-preview-image-generation",
});

const result = await model.generateContent({
  contents: [{ role: "user", parts: [{ text: imagePrompt }] }],
  generationConfig: { responseModalities: ["IMAGE", "TEXT"] },
});

// Extract image from result and convert to Buffer for upload
```

The image should always be 1200×630px (ideal OG format) or as close as the API supports, saved as WebP for performance.

## AUTOMATIC INTERNAL LINKING

When creating a post, find related posts by:

1. Same `topic_cluster`
2. Overlapping keywords
3. Adjacent funnel stage (TOP → MIDDLE → BOTTOM)

Save IDs in `related_post_ids` and render in the `RelatedPosts` component.

## SMART CTAs BY FUNNEL STAGE

The `CTABanner` component should vary by `funnel_stage`:

- **TOP:** Soft CTA — "Discover tools that make your life easier" → link to MIDDLE post or features page
- **MIDDLE:** Medium CTA — "Compare the best options on the market" → link to BOTTOM post or pricing
- **BOTTOM:** Strong CTA — "Start free now" → direct link to `${WEBSITE_URL}/signup`

## ISR AND PERFORMANCE

- Post pages: `export const revalidate = 604800` (7 days)
- Homepage: `export const revalidate = 3600` (1 hour)

Mandatory optimizations for a 100 Google PageSpeed score:

- `next/image` with `priority` on LCP (cover image from Gemini)
- `next/font` for fonts without layout shift
- No render-blocking external CSS
- Third-party components with `dynamic(() => import(...), { ssr: false })` when needed
- Prefetch internal routes via `<Link>`
